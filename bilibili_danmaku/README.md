# Bilibili 弹幕插件

实时监听 B站直播间弹幕，过滤后推送给 AI 分析。

## 功能

- 🔐 **零配置登录**：自动读取 NEKO 已保存的 B站 Cookie，无需重复输入
- 👤 **游客模式**：未登录时以游客身份监听，支持基础敏感词过滤
- 🔒 **高级过滤**（登录用户）：
  - 用户等级过滤（过滤低等级用户弹幕）
  - 礼物/SC 价值过滤（只推送高价值互动）
- 🚫 **敏感词过滤**：自动屏蔽政治敏感、暴力色情、违法违规、低俗内容
- ⏱️ **可调间隔**：推送频率 10s ~ 3min 自由设定
- 📊 **格式化输出**：弹幕按类型分组，AI 可直接读取

## 安装

将本目录放置于 `plugin/plugins/bilibili_danmaku/`，NEKO 将自动加载。

**依赖**：需要安装 `cryptography`（用于凭据加密）和 `aiohttp`、`websockets`（NEKO 原生支持，通常已安装）。本插件**不**依赖 `bilibili-api-python`，使用纯 WebSocket 实现。

## 入口说明

| 入口 | 说明 |
|------|------|
| `set_room_id` | 设置/切换直播间（必填，传入直播间数字ID） |
| `set_interval` | 调整推送间隔（10~180秒） |
| `get_danmaku` | 获取最新弹幕（AI 主动拉取） |
| `get_status` | 查看插件运行状态 |

## 使用流程

### 1. 启动后自动连接

如果 `data/config.json` 中配置了 `room_id`，插件启动时会自动连接。

### 2. 手动设置直播间

```
调用 set_room_id，参数 room_id = 1234567
```

### 3. AI 获取弹幕

```
调用 get_danmaku
```

## 配置文件

### `data/config.json` — 基础配置

```json
{
  "room_id": 1234567,
  "interval_seconds": 30
}
```

- `room_id`：直播间ID，0 表示不自动连接
- `interval_seconds`：AI 推送间隔（10~180秒）

### `data/filter_config.json` — 过滤器配置（登录用户专属）

```json
{
  "filter_level_enabled": true,
  "min_user_level": 10,
  "filter_gift_enabled": true,
  "min_gift_value": 1.0
}
```

| 字段 | 说明 |
|------|------|
| `filter_level_enabled` | 是否过滤低等级用户 |
| `min_user_level` | 最低用户等级（0-60） |
| `filter_gift_enabled` | 是否过滤低价值礼物/SC |
| `min_gift_value` | 最低礼物价值（人民币） |

> **注意**：过滤配置仅在 NEKO 已登录 B站账号时生效，游客模式只有基础敏感词过滤。

## 敏感词过滤规则

插件内置以下类别的过滤规则：

1. **政治敏感**：颠覆政权、邪教、极端主义等
2. **暴力色情**：色情、自残、毒品、武器等
3. **违法违规**：赌博、诈骗、传销等
4. **低俗粗口**：常见粗口词汇及变体
5. **垃圾信息**：拉群、涨粉等广告内容

被过滤的弹幕**不会**出现在推送给 AI 的内容中，也不会显示在日志里（仅 DEBUG 级别记录过滤原因）。
